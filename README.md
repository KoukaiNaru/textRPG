Laravel textRPG (Core Mechanics)
Проект разработан в рамках перехода в сферу бэкенд-разработки.
Цель — не просто использовать возможности Laravel «из коробки», а детально изучить внутренние механизмы фреймворка, работу с реляционными базами данных (PostgreSQL) и управление состоянием пользователя.

Ключевые архитектурные решения:

    Custom Session Management: Вместо стандартных Starter Kits (Breeze/Jetstream) реализована собственная логика аутентификации на базе нативных сессий Laravel.
    Это позволило глубже изучить жизненный цикл запроса и механизм Middleware.
    
    Data Integrity & Security: Связи между сущностями (User — Item) реализованы через Eloquent Relationships (One-to-Many).
    Безопасность данных обеспечивается на уровне контроллеров: доступ к ресурсам строго ограничен владельцем записи, что предотвращает ID-guessing атаки.
    
    Database Schema: Структура таблиц спроектирована с учетом целостности данных (foreign keys, cascade deletes).
    Миграции используются для версионирования структуры БД.
    
    Clean Code Principles: В ходе рефакторинга применен принцип DRY.
    Вспомогательные методы в контроллерах и цепочки вызовов Eloquent оптимизируют количество запросов к БД.

Технический стек:

    Language: PHP 8.x
    Framework: Laravel 11
    Database: PostgreSQL
    Frontend: Blade Templates (динамический рендеринг состояний)

Статус проекта:
Реализован базовый цикл CRUD для инвентаря игрока. В процессе разработки — оптимизация обработки ошибок и внедрение системы уведомлений.
Проект служит демонстрацией навыков работы с документацией, отладки окружения и проектирования логики «с нуля».
