# Laravel textRPG (Inventory System)

Учебный проект для отработки навыков работы с MVC-архитектурой и базой данных PostgreSQL. Основная цель — понять, как Laravel управляет потоками данных «под капотом».

### Что реализовано:
*   **Custom Auth Logic:** Работа с сессиями PHP для идентификации пользователей без использования встроенных систем (Breeze/Jetstream).
*   **Inventory CRUD:** Создание, чтение и просмотр игровых предметов через Eloquent ORM.
*   **Database Management:** Проектирование структуры таблиц через миграции и работа в связке с PostgreSQL.
*   **Validation:** Валидация входящих запросов в контроллерах для обеспечения целостности данных.

### Стек:
*   **PHP 8 / Laravel 11**
*   **PostgreSQL**
*   **Blade Templates**
*   **Git / GitHub**

### Технические вызовы, с которыми я справился:
1. **Настройка окружения:** Отладка подключения к PostgreSQL (127.0.0.1) и решение проблем с системными прокси.
2. **Маршрутизация:** Организация связей между роутами, контроллерами и методами модели.
3. **Data Integrity:** Реализация уникальности имен пользователей на уровне базы данных и валидатора.
4. **Git Workflow:** Опыт работы с ветками, разрешение конфликтов при первом деплое проекта на GitHub.

### Как запустить локально:
1. `git clone https://github.com`
2. `composer install`
3. Настроить подключение к БД в `.env` файле.
4. `php artisan migrate`
5. `php artisan serve`
